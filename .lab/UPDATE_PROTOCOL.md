# UPDATE_PROTOCOL

## Fluxo

1. Ler README.md.
2. Ler INDEX.md.
3. Identificar os documentos relacionados ao assunto.
4. Ler esses documentos.
5. Comparar o conteúdo da conversa com o conhecimento existente.
6. Integrar apenas conhecimento novo.
7. Atualizar somente os documentos afetados.
8. Nunca recriar documentos já existentes sem antes entendê-los.
9. Gerar arquivos completos (não apenas trechos).
10. O usuário é responsável por copiar os arquivos para o clone local e executar git add/commit/push.

## Regras

- Nunca atualizar baseado apenas na conversa.
- O repositório é a fonte oficial.
- Atualizar significa fazer merge, não reescrever.
